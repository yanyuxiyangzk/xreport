## 一、产品简介

**XReport** 是一款面向开发者和企业的**拖拽式可视化报表设计器**，支持多数据源接入、SQL 查询聚合、图表可视化以及 Excel / PDF / Word 多格式导出。

核心特点：
- 🎯 **零编码** - 拖拽即可完成报表设计
- 🔗 **多数据源** - JDBC 多数据库 + REST API
- 📊 **丰富图表** - Chart.js 图表 + Echarts 仪表盘
- 📤 **多格式导出** - Excel / PDF / Word
- 🔐 **权限管控** - Spring Security + JWT + 细粒度 RBAC
- 🚀 **开源可控** - 完全开源，支持二次开发

## 二、核心功能模块

### 2.1 报表设计器（Phase 2 ✅）

| 功能 | 说明 |
|------|------|
| Luckysheet 在线编辑 | 类 Excel 的在线表格编辑器，支持单元格合并、公式、样式 |
| 多 Sheet 管理 | 一个报表模板支持多个 Sheet |
| 拖拽组件面板 | 字段、表格、图表、指标卡等组件 |
| 组件属性配置 | 支持编辑列配置、图表配置、指标卡数值和位置/尺寸 |
| 双击/拖拽添加组件 | 组件可双击或拖拽到画布 |

**技术实现**：Luckysheet + Vue 3 + ElementPlus

---

### 2.2 数据源管理（Phase 3 ✅）

| 功能 | 说明 |
|------|------|
| JDBC 多数据库 | 支持 MySQL / PostgreSQL / Oracle / SQLServer 等 |
| API 数据源 | 支持 REST API 接口数据接入 |
| 连接测试 | 一键测试 JDBC / API 连接是否正常 |
| 数据源启用/禁用 | 管理数据源状态 |

**API 接口**：
```
POST /api/datasources/test/jdbc  - 测试 JDBC 连接
POST /api/datasources/test/api  - 测试 API 连接
POST /api/datasources/query/jdbc - 执行 JDBC SQL 查询
POST /api/datasources/query/api  - 执行 API 查询
```

**技术实现**：Druid 连接池 + Spring RestTemplate

---

### 2.3 数据集管理（Phase 4 ✅）

| 功能 | 说明 |
|------|------|
| SQL 查询 | 可视化 SQL 查询构建 |
| 聚合计算引擎 | 支持 SUM / AVG / COUNT / MAX / MIN 等聚合 |
| 数据集 CRUD | 创建、编辑、删除数据集 |
| 多数据源关联 | 支持跨数据源查询 |

**API 接口**：
```
POST /api/datasets/aggregate - 聚合计算
GET/POST/PUT/DELETE /api/datasets - 数据集 CRUD
```

**技术实现**：MyBatis-Plus + JdbcUtils

---

### 2.4 报表预览与导出（Phase 5 ✅）

| 导出格式 | 说明 |
|------|------|
| Excel (.xlsx) | 使用 Apache POI 导出，包含真实数据 |
| PDF | 矢量格式，支持打印 |
| Word (.docx) | 支持套用模板 |
| CSV | 数据导出 |

**API 接口**：
```
POST /api/export/excel - 导出 Excel
POST /api/export/pdf   - 导出 PDF
POST /api/export/word  - 导出 Word
POST /api/report/render - 渲染报表预览
```

**技术实现**：Apache POI + jsPDF + xlsx (SheetJS)

---

### 2.5 Word 模板生成（Phase 6 ✅）

| 功能 | 说明 |
|------|------|
| 模板变量替换 | 支持 `{{fieldName}}` 语法 |
| 条件渲染 | 支持 `{{#if condition}}` 条件块 |
| 循环渲染 | 支持 `{{#each items}}` 循环块 |
| 图片嵌入 | 支持将图表/图片嵌入 Word |

**技术实现**：poi-tl 模板引擎

---

### 2.6 仪表盘 / 大屏设计（Phase 7 ✅）

| 功能 | 说明 |
|------|------|
| 拖拽布局 | 可视化拖拽设计仪表盘 |
| 图表类型 | 折线图、柱状图、饼图、散点图（Chart.js） |
| 实时数据 | 支持刷新频率配置 |
| 自适应大屏 | 适配各种屏幕分辨率 |

**技术实现**：Vue 3 + Chart.js + Echarts

---

### 2.7 系统管理与权限（Phase 1 ✅ / Phase 8 ✅）

| 功能 | 说明 |
|------|------|
| 用户管理 | 增删改查、状态启用/禁用 |
| 角色管理 | 角色 CRUD，角色权限分配 |
| 部门管理 | 组织架构树形管理 |
| 菜单管理 | 前端路由 + 权限标识 |
| JWT 认证 | Token 无状态认证 |
| 细粒度 RBAC | 菜单级 + 按钮级权限控制 |

**默认账号**：`admin` / `admin123`

**API 接口**：
```
POST /api/auth/login          - 登录
GET/POST/PUT/DELETE /api/system/users  - 用户管理
GET/POST/PUT/DELETE /api/system/roles  - 角色管理
GET/POST/PUT/DELETE /api/system/depts  - 部门管理
GET/POST/PUT/DELETE /api/system/menus - 菜单管理
```

**技术实现**：Spring Security + Spring Security Crypto (BCrypt) + jjwt

## 三、技术架构

### 3.1 整体架构

```
┌─────────────────────────────────────────────────────────┐
│                      前端 (Vue 3)                        │
│  报表设计器 │ 仪表盘 │ 报表预览 │ 系统管理 │ 分享查看      │
└──────────────────────┬──────────────────────────────────┘
                        │ HTTP / REST API
┌──────────────────────▼──────────────────────────────────┐
│                    后端 (Spring Boot 3)                   │
│  Controller │ Service │ Mapper │ POJO │ Common            │
└──────────────────────┬──────────────────────────────────┘
                        │

│                   数据存储层                              │
```

### 3.3 后端模块结构（Maven 多模块）

```
backend/
├── xreport-web/          # Controller 层，接收 HTTP 请求
├── xreport-service/      # Service 层，业务逻辑处理
├── xreport-mapper/       # Mapper 层，数据库访问
├── xreport-pojo/         # POJO / DTO / VO，数据对象
├── xreport-common/       # 公共工具类、枚举、异常
├── docs/                 # 部署文档、监控文档
├── scripts/             # 部署脚本
├── sql/                  # 数据库初始化脚本
└── docker/               # Docker 配置
```

### 3.4 前端目录结构

```
frontend/src/
├── api/                  # Axios 封装，API 请求
├── router/              # Vue Router 路由配置
├── stores/              # Pinia 状态管理
├── views/
│   ├── report/          # 报表设计器、列表、预览
│   ├── dashboard/       # 仪表盘
│   ├── datasource/      # 数据源管理
│   └── share/           # 分享查看
├── components/          # 公共组件
├── utils/               # 工具函数
└── assets/             # 静态资源
```

## 六、部署方式

### 6.1 Docker 部署（推荐）

```bash
# 前端 + 后端一键启动
docker-compose up -d

# 仅启动后端
docker build -t xreport-backend .
docker run -d -p 8080:8080 xreport-backend
```

### 6.2 手动部署

**后端**：
```bash
cd backend
mvn clean package -DskipTests
java -jar xreport-web/target/xreport-web.jar
```

**前端**：
```bash
cd frontend
npm install
npm run build
# 部署 dist 目录到 Nginx
```

### 6.3 环境要求

| 环境 | 要求 |
|------|------|
| JDK | 17+ |
| Node.js | 18+ |
| MySQL | 8.0+ |
| Maven | 3.8+ |
| Nginx | 1.20+ |
