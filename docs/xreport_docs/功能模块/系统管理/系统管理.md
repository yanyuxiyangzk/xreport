# 系统管理

## 概述

系统管理模块提供用户、角色、部门、菜单的基础管理功能，支持灵活的权限控制。

## 用户管理

### 功能说明

管理系统的使用账户，包括用户信息的增删改查、状态启用/禁用等。

### 操作路径

菜单路径：`系统管理 → 用户管理`

### 主要字段

| 字段 | 说明 | 备注 |
|------|------|------|
| 用户名 | 登录账号 | 唯一，不可重复 |
| 密码 | 登录密码 | BCrypt 加密存储 |
| 昵称 | 显示名称 | |
| 手机号 | 联系方式 | |
| 邮箱 | 电子邮件 | |
| 部门 | 所属部门 | 树形结构 |
| 角色 | 分配角色 | 支持多角色 |
| 状态 | 启用/禁用 | 禁用后不可登录 |

### 默认账户

安装后系统自带管理员账户：

| 用户名 | 密码 | 角色 |
|--------|------|------|
| admin | admin123 | 超级管理员 |

## 角色管理

### 功能说明

管理角色定义，每个角色可分配不同的菜单权限。

### 主要字段

| 字段 | 说明 |
|------|------|
| 角色名称 | 如"报表管理员"、"普通用户" |
| 角色编码 | 唯一标识，如 `ROLE_REPORT_ADMIN` |
| 显示顺序 | 菜单中的排序 |
| 备注 | 角色描述 |

### 权限分配

角色创建后，可通过"分配菜单"功能设置该角色可访问的菜单和操作按钮。

## 部门管理

### 功能说明

树形结构的部门/组织管理，支持多层级。

### 主要字段

| 字段 | 说明 |
|------|------|
| 部门名称 | 如"研发部"、"财务部" |
| 父级部门 | 上级部门（顶级为根） |
| 显示顺序 | 同级排序 |
| 负责人 | 部门负责人姓名 |

## 菜单管理

### 功能说明

配置系统左侧菜单导航，支持多层级菜单。

### 菜单类型

| 类型 | 说明 |
|------|------|
| 目录 | 菜单分组，无对应页面 |
| 菜单 | 有对应页面的菜单项 |
| 按钮 | 页面内的操作按钮（如"新增"、"编辑"） |

### 主要字段

| 字段 | 说明 |
|------|------|
| 菜单名称 | 显示名称 |
| 父级菜单 | 上级菜单 |
| 路由路径 | 前端路由地址 |
| 组件路径 | Vue 组件路径 |
| 图标 | 菜单图标 |
| 显示顺序 | 菜单排序 |
| 权限标识 | 按钮级别权限，如 `system:user:add` |

### 菜单图标

系统使用 Ant Design 内置图标，填写图标名称即可，如 `UserOutlined`、`SettingOutlined`。

## 认证机制

### 登录

用户提交用户名和密码，后端验证后返回 JWT Token。

**请求：**
```
POST /api/auth/login
Content-Type: application/json

{
  "username": "admin",
  "password": "admin123"
}
```

**响应：**
```json
{
  "code": 200,
  "data": {
    "token": "eyJhbGciOiJIUzI1NiJ9...",
    "userId": 1,
    "username": "admin"
  }
}
```

### Token 使用

之后的请求需要在 Header 中携带 Token：

```
Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...
```

### Token 刷新

Token 有效期为 24 小时，快过期时前端自动调用刷新接口。

### 退出登录

```
POST /api/auth/logout
Authorization: Bearer <token>
```

## 权限控制

### 页面权限

通过菜单管理控制用户可见的菜单。未分配菜单的用户登录后无任何菜单。

### 按钮权限

使用权限标识（Permission）控制页面内按钮的显示/隐藏：

```typescript
// 权限控制示例
{hasPermission('system:user:add') && <Button>新增</Button>}
```

### 数据权限（规划中）

规划支持行级数据权限控制，即用户只能看到自己部门的数据。
