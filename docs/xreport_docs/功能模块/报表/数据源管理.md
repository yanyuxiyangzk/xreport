# 数据源管理

## 概述

数据源是报表的数据来源，定义了数据库连接信息。XReport 通过 JDBC 连接各种关系型数据库，执行用户编写的 SQL 语句获取数据。

## 操作路径

菜单路径：`报表管理 → 数据源管理`

## 添加数据源

点击「新增数据源」，填写以下信息：

### 基本信息

| 字段 | 说明 | 必填 |
|------|------|------|
| 数据源名称 | 标识名称，如"生产库" | 是 |
| 数据源编码 | 唯一编码，如 `ds_prod` | 是 |
| 数据库类型 | MySQL / PostgreSQL | 是 |

### 连接信息

**MySQL 示例：**

```
jdbc:mysql://localhost:3306/xreport?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
```

| 字段 | 说明 |
|------|------|
| JDBC URL | 完整 JDBC 连接地址 |
| 用户名 | 数据库用户名 |
| 密码 | 数据库密码（加密存储） |
| 连接超时 | 超时时间（默认 30 秒） |

**PostgreSQL 示例：**

```
jdbc:postgresql://localhost:5432/xreport
```

### 连接测试

填写完成后，点击「测试连接」验证配置是否正确。测试通过后再保存。

## 数据源管理

### 启用/禁用

禁用的数据源在报表中不可选用，但配置信息保留。

### 编辑

修改后需要重新测试连接。

### 删除

删除前检查是否有报表正在使用该数据源。被引用时不可删除。

## 数据源安全

### 密码加密

所有数据库密码在存储前使用 AES 加密，不以明文形式保存。

### SQL 注入防护

系统对所有 SQL 参数使用预编译（PreparedStatement）方式执行，防止 SQL 注入攻击。

### 权限控制

数据源管理需要 `system:datasource` 权限，普通用户无法访问。

## 多数据源场景

### 场景一：生产库 + 历史库

报表需要同时查询生产数据和历史归档数据，可分别配置两个数据源，在报表中使用多个数据集分别查询。

### 场景二：主从分离

配置两个数据源分别指向主库和从库，报表查询使用从库以减轻主库压力。

## 常见问题

### 1. 连接测试失败

**检查项：**
- MySQL/PostgreSQL 服务是否启动
- 端口是否正确（MySQL 默认 3306，PostgreSQL 默认 5432）
- 用户名密码是否正确
- 目标数据库是否已创建
- 是否有防火墙阻挡

### 2. 连接超时

增加连接超时时间，或检查网络到数据库服务器是否稳定。

### 3. 中文乱码

确保 JDBC URL 中指定了字符编码：

```
jdbc:mysql://localhost:3306/xreport?characterEncoding=utf8
```
